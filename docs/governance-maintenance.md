# Governança, manutenção e contribuição

OwnPaper é um projeto aberto, mas não deve ser entendido como um produto com manutenção contínua garantida.

## Origem do projeto

O OwnPaper foi desenvolvido em conjunto com OpenAI Codex a partir de necessidades reais de uma instalação editorial específica. Isso acelera a construção, mas não elimina a necessidade de revisão humana, testes e validação operacional antes de usar em produção.

## Política de manutenção

O mantenedor original pretende priorizar:

- recursos necessários nas próprias instalações;
- correções de bugs que afetem essas instalações;
- ajustes de segurança e privacidade relevantes para o uso atual;
- documentação e empacotamento que facilitem reinstalação e operação.

Não há compromisso de roteiro público, SLA de resposta, suporte comercial, compatibilidade indefinida ou manutenção contínua de recursos solicitadas por terceiros.

## Forks

Se alguém quiser continuidade mais ampla, outra direção de produto, suporte a outros fluxos ou evolução comunitária, o caminho recomendado é criar um fork.

A licença MIT permite esse uso com pouca restrição, desde que os avisos de licença sejam preservados.

## Contribuições

Contribuições podem ser úteis, mas devem vir acompanhadas de testes e documentação quando alterarem comportamento. Pull requests podem ficar sem resposta por tempo indeterminado.
